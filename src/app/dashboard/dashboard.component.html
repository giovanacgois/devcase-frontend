Dashboard

O Dashboard ainda não foi implementado. Por isso, nesta página, será mostrado um resumo do que foi desenvolvido no tempo proposto.

O objetivo: O esperado da guia Dashboard é que mostrasse ao usuário a visão geral geral dos clientes cadastrados, suas pontuações e últimas vendas cadastradas; visualização de informações e aplicação de filtros de busca.

Clientes

Funcionalidades Implementadas: Recebe os dados em formulário; Armazena registros; Exibe listagem geral. Os dados do formulário são validados automaticamente (Client-Side) e dicas são exibidas ao usuário para auxiliá-lo no preenchimento.

Funcionalidades Pendentes: Alterar registro; Excluir registro.

Melhorias Futuras: Implementar validador de CPF, e-mail e CEP; importar lista de cidades e UF para seleção via menu drop-down, facilitando a experiência do usuário e evitando erros de digitação por parte do usuário.

Regras de Pontuação

Funcionalidades Implementadas: A estrutura foi iniciada no Back-End, mas não foram concluídas nessa Sprint.

Funcionalidades Pendentes: Cadastrar Pontuação via formulário; Alterar registro; Excluir registro; Listar regras ativas.

Ideia de Implementação: Receber os dados de pontuação via "range", alterando automaticamente os valores máximos e mínimos de acordo com o que cadastrado. Ex.: [Regra 1] = Usuário seleciona Valor Mínimo=R$0 e Valor máximo=R$50; então, automaticamente, cria-se a [Regra 2] com valor mínimo=R$50,01, com valor máximo e pontuação a ser definida pelo usuário.

Vendas

Funcionalidades Implementadas: A estrutura foi iniciada no Back-End, mas não foram concluídas nessa Sprint. Há o formulário para cadastro, porém não está funcional.

Funcionalidades Pendentes: Integrar funcionalidade à API do PDV da VSM, recebendo dados das vendas realizadas a atualizando automaticamente as pontuações.
